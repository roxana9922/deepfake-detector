# Screenshots

This folder contains screenshots of the demonstration artefact.

These screenshots are used as evidence that the interface was developed and that the main workflow was tested.

Expected screenshots include:

- `upload_interface.png`
- `prediction_result.png`
- `gradcam_output.png`
- `invalid_input_example.png`

The screenshots should show the user interface, the uploaded image, the prediction result, and the Grad-CAM explanation. They can also show how the system handles incorrect or unsupported inputs.

This helps demonstrate that the project includes a working software artefact, not only a written report.
